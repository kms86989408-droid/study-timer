# Product Context

## One-Line Summary

study-timer는 개인이 공부 시간과 휴식 시간을 5분 단위로 설정하고, 세션 기록을 캘린더로 확인하며, Google 계정으로 데이터를 동기화하는 PWA다.

## Users

- 개인 공부 시간을 꾸준히 기록하려는 사용자
- 포모도로식 공부/휴식 루틴을 원하는 사용자
- 추후 모바일 앱으로 같은 데이터를 이어 쓰고 싶은 사용자

## Primary Outcomes

- 사용자가 공부 시간과 휴식 시간을 빠르게 설정하고 타이머를 시작한다.
- 타이머 진행 중 남은 시간이 시각적으로 줄어드는 이미지로 즉시 이해된다.
- 하루 단위 공부/휴식 기록이 캘린더에 누적된다.
- Google 로그인 후 개인 기록이 기기 간 동기화된다.
- MVP에서는 복잡한 분석보다 기록 신뢰성과 사용 흐름의 단순함을 우선한다.

## Non-Goals

- 1차 범위에서는 Google Calendar 또는 Google Sheets 직접 동기화를 제공하지 않는다.
- 1차 범위에서는 네이티브 앱스토어 배포를 하지 않는다.
- 1차 범위에서는 팀, 클래스, 관리자용 통계 기능을 제공하지 않는다.

## Constraints

- 시간 설정은 5분 배수만 허용한다.
- 초기 배포는 EC2 + Docker 기반으로 시작하고 프리티어 또는 신규 AWS 크레딧 범위를 고려한다.
- DB는 EC2에 직접 운영하지 않고 Firebase Auth/Firestore를 사용한다.
- 공개 API와 데이터 계층은 추후 앱 전환을 고려해 클라이언트 UI와 분리한다.
- 민감 정보와 Firebase/AWS 시크릿은 코드와 문서에 직접 저장하지 않는다.

## Current Unknowns

- 줄어드는 이미지의 구체적 컨셉과 에셋 방식
- Google 로그인 전 로컬 기록의 동기화 충돌 정책
- 앱 전환 시점의 대상 기술: React Native, Expo, 또는 WebView 기반 패키징
