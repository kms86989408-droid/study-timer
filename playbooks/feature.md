# Feature Playbook

## Goal

새 기능을 기존 시스템에 안전하게 추가한다.

## Intake Checklist

- 사용자 요구는 무엇인가
- 성공 조건은 무엇인가
- 기존 동작 중 보존해야 할 것은 무엇인가
- 관련 모듈, API, 화면, 데이터 스키마는 무엇인가

## Execution Flow

1. 기존 구현과 유사 사례를 찾는다.
2. 변경 범위를 입력, 처리, 출력 관점으로 나눈다.
3. 인터페이스 변경 여부를 확인한다.
4. 가장 작은 구현 단위부터 추가한다.
5. 테스트와 문서를 함께 갱신한다.

## Deliverables

- 기능 코드
- 필요한 테스트
- 사용 방법 또는 제약 문서
- 남은 리스크
