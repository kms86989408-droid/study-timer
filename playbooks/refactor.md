# Refactor Playbook

## Goal

외부 동작을 유지한 채 구조를 개선한다.

## Preconditions

- 리팩터링의 이유가 분명한가
- 유지해야 할 동작이 정의되어 있는가
- 검증 수단이 있는가

## Execution Flow

1. 바꾸지 않을 외부 동작을 명시한다.
2. 변경 전후를 비교할 검증 수단을 고른다.
3. 구조 변경을 작은 단계로 나눈다.
4. 각 단계마다 동작을 다시 확인한다.
5. 기능 변경이 섞이기 시작하면 작업을 분리한다.

## Common Targets

- 긴 함수 분리
- 책임 이동
- 중복 제거
- 이름 정리
- 모듈 경계 정리
