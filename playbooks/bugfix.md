# Bugfix Playbook

## Goal

버그의 원인을 좁히고, 재발 가능성을 줄이는 최소 수정으로 해결한다.

## Triage

- 기대 동작과 실제 동작을 분리한다.
- 재현 조건을 가능한 한 구체적으로 적는다.
- 최근 변경, 입력 데이터, 환경 차이를 확인한다.

## Execution Flow

1. 재현 경로를 확보한다.
2. 원인을 좁힌다.
3. 증상만 가리는 임시 수정인지 확인한다.
4. 최소 수정으로 원인을 제거한다.
5. 같은 유형의 회귀가 없는지 주변 경로를 확인한다.

## Guardrails

- 추측만으로 넓게 고치지 않는다.
- 재현이 안 되면 관측 신호를 먼저 늘린다.
- 가능하면 회귀 테스트 또는 재현 절차를 남긴다.
