# Review Playbook

## Goal

변경사항의 위험과 누락을 빠르게 찾는다.

## Review Method

1. 변경 의도를 파악한다.
2. 영향 받는 사용자 흐름을 찾는다.
3. 실패 조건을 먼저 상상한다.
4. 테스트가 그 실패를 막는지 본다.
5. 남은 리스크를 우선순위 순으로 정리한다.

## Findings Format

- 심각도
- 파일 또는 위치
- 문제 설명
- 왜 중요한지
- 가능한 수정 방향 또는 검증 아이디어

## Review Biases To Avoid

- 스타일 문제에 먼저 매달리지 않는다.
- 익숙한 코드라는 이유로 가정을 생략하지 않는다.
- 설명만 보고 승인하지 않고 실제 변경 내용을 읽는다.
