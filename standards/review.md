# Review Standards

## Review Priorities

코드 리뷰는 아래 순서로 본다.

1. 동작이 맞는가
2. 회귀 위험이 있는가
3. 테스트가 충분한가
4. 유지보수성이 나빠지지 않는가
5. 스타일 문제는 중요한 이슈 뒤로 미룬다

## Severity Guide

- Critical: 데이터 손실, 보안 문제, 장애 가능성
- High: 사용자 기능 오동작, 큰 회귀 가능성
- Medium: 특정 조건에서의 버그, 테스트 누락, 설계 부채
- Low: 가독성, 일관성, 작은 개선 포인트

## Review Output

- 문제를 먼저 말한다.
- 파일 또는 위치를 함께 적는다.
- 왜 문제가 되는지와 어떤 조건에서 드러나는지 설명한다.
- 테스트 또는 재현 아이디어를 붙이면 더 좋다.
