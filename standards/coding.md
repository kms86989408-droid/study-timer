# Coding Standards

## Core Rules

1. 기존 패턴을 먼저 따른다. 새 추상화는 반복이 확인된 뒤에만 도입한다.
2. 한 변경은 하나의 의도를 가져야 한다.
3. 이름은 도메인 의미를 드러내야 한다.
4. 불필요한 전역 영향 변경을 피한다.
5. 복잡한 로직에는 짧은 설명 주석을 허용하되, 당연한 설명은 쓰지 않는다.

## Change Strategy

- 먼저 가장 작은 수정으로 문제를 해결할 수 있는지 확인한다.
- 공개 인터페이스를 바꿀 때는 영향 범위를 적는다.
- 설정값, 플래그, 환경변수 추가는 기본값과 실패 시 동작을 함께 정의한다.

## Safety

- 사용자가 만든 변경을 임의로 되돌리지 않는다.
- 대규모 포맷 변경은 기능 변경과 분리한다.
- 민감 정보나 비밀값은 문서나 코드에 직접 넣지 않는다.

## Documentation

- 새로운 개념이나 운영 규칙이 생기면 관련 문서를 함께 갱신한다.
- 사용 방법이 바뀌면 PR 설명이나 템플릿도 같이 갱신한다.
